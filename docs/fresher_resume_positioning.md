# Fresher Resume Positioning

Use this project as one high-signal portfolio project for on-campus Data Analyst, Business Analyst, Product Analyst, and Analytics Intern roles.

## Best Project Title

`QuickCommerceOps: Stockout, Replenishment and Product Analytics`

## One-Line Resume Entry

Built an end-to-end quick-commerce analytics project using SQL, Python, Excel, and Power BI-ready outputs to analyze stockout risk, category health, replenishment priorities, data-quality gaps, and Product Analyst scenario metrics across 3.7K+ SKU inventory rows.

## Best Resume Bullets

- Cleaned and modeled 3.7K+ SKU inventory rows using Python and SQL, handling encoding issues, paise-to-rupee conversion, invalid commercial records, duplicate-like rows, and category-level KPI creation.
- Built reusable SQL views and Python pipelines for category health, discount exposure, replenishment watchlists, data-quality audits, and Excel/Power BI-ready exports.
- Designed a transparent Product Analyst scenario layer to practice fill rate, stockout order rate, unfulfilled value, substitution acceptance, what-if recovery, and experiment design without claiming simulated metrics as actual sales.

## Short Interview Pitch

This project starts as a Data Analyst inventory project and extends into Product Analyst thinking. The observed dataset contains SKU inventory, so I used it for stockout rate, category health, discount exposure, and replenishment priority. Since it does not contain real orders, I created a clearly labeled scenario layer to demonstrate how I would evaluate fill rate, unfulfilled value, substitutions, and experiments if order data were available.

## Best Roles To Use It For

- Data Analyst
- Business Analyst
- Product Analyst
- Growth Analyst Intern
- Operations Analyst
- Supply Chain Analytics Intern
- Category Analyst

## What To Avoid

- Do not call the scenario file actual sales data.
- Do not say the project predicts demand.
- Do not say the unfulfilled value is actual lost revenue.
- Do not over-focus on tools. Lead with business problem and decision.

## Best Closing Line In Interviews

The main thing I learned is that analytics is not just creating charts. It is also knowing what the data can and cannot prove, separating observed facts from assumptions, and turning the result into a decision a team can act on.
